What's CSS?

CSS is a programing language used to make HTML (HyperText Markup Language) less plain and nicer looking. This may sound unimportant, but you can do amazing things with CSS.

Ok, what's HTML?

HTML is a programming language that is used to code webpages with tags. Right now when I'm typing this paragraph in my own webpage, I'm typing it between an opening and closing tag that look like this:

<p>HTML is a programming...</p>

How do I use CSS?

To use CSS you need to have a selector. A selector is something you use to identify one or more HTML tags. There are many selectors and as many ways to use them, so I'll only explain a few. I will explain basic methods of using tag names, ids, and classes

Style Tags

You need a style tag just as much as you need a selector. A style tag lets you use CSS by telling the computer you switched the programming languages. The computer would give you syntax errors while you were using valid CSS A style tag is exactly the same as a paragraph tag, except for the "p" is replaced with "style". This is a style tag:

<style>(CSS)</style>

note: (CSS) would be replaced with actual CSS code.

Tag Names as Selectors

This is by far the easiest selector to use. Simply type the tag name of your choice inside the style tag and put a pair of brackets after it, like this:

<style>h1 {}</style>

*note: h1 is a tag used for headlines. It is basically a paragraph tag with larger text.

The main disadvantage to this method is that it selects all tags of the type you select and just that type. It isn't very fexible.

Ids

An id is th best tool you have if you only want to select one element. To use an id as a selector:

  1. Put an id atribute* in the opening element of your tag
  2. Type "#id-name {}" in your style tag

*note: An atribute is used to specify something about a tag that isn't specified in the tag name.

*note: Id-name refers to the name of your id. You may use anything as the name of your id as long as it has no spaces. This also applies to class names.

<style>#my-id {}</style>...
<p id="my-id">Whatever I want to type</p>

Classes

If you use tag names as selectors, you use all tags with the name you selected, and no other tags. If you use a id as selector, you can only select one tag. Classes are the best of both worlds. Classes are like ids but you can give multible tags the same class and you replace the hashtag (#) with a period. An exmple:

<style>.my-class {}</style>...
<p class="my-class">A paragraph</p>
<h1 class="my-class">A headline</h1>

CSS Syntax

But how do you actually do stuff with CSS? Well, you use properties! Properties are what actually specify how the element looks. For example, to specify the color of the background of this text, I use this property:

background-color: rgba(250, 128, 114, 0.60);

You might not understand properties now, but keep reading for more information.

The syntax of properties

Look up, above the headline above the paragraph. You saw it, right? Up there is a small segment of code that looks like it would tell the computer to make the background of some element a rgba color. Seems like it would work the same if you typed one of these:

background-color: rgba(250, 128, 114, 0.60)
background-color rgba(250, 128, 114, 0.60);
background-color: rgba(250 128 114 0.60);

Nope! Not at all. When you're typing a property, you need to follow the rules of CSS syntax. CSS syntax looks like:

property-name: property-values(perameters);

*note: I am not sure what the technical term for "perameters" is.

*note: I am not sure what the technical term for "property-values" is.

Now, let's dissect the syntax of the property above.

  • property-name:
    This refers to the name of the property. For example: "border-style" or "text-align". After the property name is a semicolon (;)
  • property-values:
    This represents the value(s)* of the property. Since so far it might have been a little confusing, i'll give you a metaphor for CSS syntax. The metaphor is that the property name is a question and the value is the answer. For example: "What size should the margin be? it should be 10 pixels."
  • perameters:
    As you can see, the perameters are in parentheses(you know what parentheses are, right?). The other puncuation mark for perameters is the comma (,), which seperates perameters if there are more than 1 of them. Before we get into detail, it is important to note that:
    • There can be nesting perameters (perameters inside perameters)
    • If there are multiple property values, there can be multiple sets of perameters.

Perameters are generally used to specify more about a property value. To continue with our question example, it would sound a little like this: "What color should the background be? An RGB color. What is the number for the red? 255. What is the number for the green? 10. What is the number for the blue? 10."
